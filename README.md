# Game-Design

This repository documents my game design learning and production process from the University of Washington Game Design Certificate program.

I will use Unity to build playable prototypes and demos, including:
- 2D games (core loop, interaction design, level design, and polish)
- 3D games (world interaction, camera/game feel, and gameplay systems)

The goal of this repository is to connect design thinking with real implementation: from concept notes and mechanics documents to playable Unity projects.

## Repository Structure

### Unity_Game_Demo
Unity project work and playable prototypes. This folder is for implementation, iteration, and technical validation of design ideas.

### Game_design_doc
Design documentation hub. This folder stores ideas, rules, and design breakdowns before and during implementation.

Current subfolders in Game_design_doc:
- 2D_game: design notes and mechanics planning for 2D projects.
- 3D_game: design notes and mechanics planning for 3D projects.
- Board_game: board game concepts, rules, and balancing notes.
- Card_game: card system design, card interactions, and deck rules.
- Monopoly_Adjust: adjustment and redesign notes based on Monopoly-style systems.
- Puzzle: puzzle mechanics, level logic, and player guidance design.
- TTPRG: tabletop RPG related world/rule design and campaign mechanics.
- VR_game: VR-specific interaction and experience design notes.

## Workflow

1. Start from design documents in Game_design_doc.
2. Build and test prototypes in Unity_Game_Demo with Unity.
3. Iterate based on playtest feedback and update both docs and implementation.

## Ongoing Plan

- Continuously convert document ideas into playable Unity prototypes.
- Improve game feel, balancing, and usability through iteration.
- Keep design docs and implementation progress aligned.
